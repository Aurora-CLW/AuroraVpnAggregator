# 功能需求文档

## 1. 功能概述

Aurora VPN Aggregator 是一个自动化的 VPN 订阅聚合系统，通过定时抓取多个订阅源，测试节点有效性，生成高质量的订阅链接。

---

## 2. 功能需求

### 2.1 订阅源管理模块

#### 2.1.1 GitHub 订阅抓取

| 需求项 | 描述 |
|--------|------|
| **功能描述** | 从 GitHub 仓库抓取订阅链接 |
| **输入** | 订阅 URL、格式类型 |
| **输出** | 订阅内容字符串 |
| **业务规则** | 1. 支持重试机制（默认 3 次）<br>2. 支持超时控制（默认 30 秒）<br>3. 失败时记录错误日志 |
| **验收标准** | 能够成功抓取配置的所有 GitHub 订阅源 |

#### 2.1.2 本地文件读取

| 需求项 | 描述 |
|--------|------|
| **功能描述** | 读取本地存储的节点文件 |
| **输入** | 文件路径、格式类型 |
| **输出** | 文件内容字符串 |
| **业务规则** | 1. 支持 YAML、TXT、Base64 格式<br>2. 支持文件变更监控（可选）<br>3. 文件不存在时跳过 |
| **验收标准** | 能够正确读取本地节点文件 |

#### 2.1.3 Telegram 频道解析

| 需求项 | 描述 |
|--------|------|
| **功能描述** | 从 Telegram 频道抓取节点分享消息 |
| **输入** | 频道 ID、最大消息数、关键词过滤 |
| **输出** | 节点 URL 列表 |
| **业务规则** | 1. 需要 API ID 和 API Hash<br>2. 支持关键词过滤<br>3. 支持消息缓存<br>4. 限流控制（请求间隔 1 秒） |
| **验收标准** | 能够从配置的 Telegram 频道提取节点 |

---

### 2.2 节点处理模块

#### 2.2.1 多格式解析

| 需求项 | 描述 |
|--------|------|
| **功能描述** | 解析多种协议格式的节点 |
| **支持的协议** | SS、SSR、VMess、VLess、Trojan、Hysteria2、TUIC、Reality |
| **支持的格式** | Clash YAML、V2Ray Base64、Sing-box JSON、原始 URL |
| **业务规则** | 1. 自动识别内容格式<br>2. 解析失败时记录错误<br>3. 提取完整的节点参数 |
| **验收标准** | 正确解析所有支持的协议格式 |

#### 2.2.2 节点去重

| 需求项 | 描述 |
|--------|------|
| **功能描述** | 识别并去除重复节点 |
| **去重算法** | 基于节点指纹（类型+服务器+端口+UUID/密码） |
| **业务规则** | 1. 生成唯一指纹<br>2. 保留首次出现的节点<br>3. 记录重复数量 |
| **验收标准** | 去重后无重复节点 |

#### 2.2.3 地理位置识别

| 需求项 | 描述 |
|--------|------|
| **功能描述** | 识别节点服务器所在国家/地区 |
| **识别方式** | GeoIP 数据库查询 |
| **输出信息** | 国家代码、国家名称、城市、ISP |
| **业务规则** | 1. 使用离线 GeoIP 数据库<br>2. 识别失败时标记为 Unknown |
| **验收标准** | 90% 以上节点能正确识别国家 |

### 2.3 节点测试模块

#### 2.3.1 TCP 可达性测试

| 需求项 | 描述 |
|--------|------|
| **功能描述** | 测试节点服务器的 TCP 端口是否可达 |
| **测试方式** | TCP 连接测试 |
| **并发数** | 100 |
| **超时时间** | 3 秒 |
| **业务规则** | 1. 使用多线程并发测试<br>2. 连接成功标记为 TCP 有效<br>3. 记录测试时间 |
| **验收标准** | TCP 不可达的节点被过滤 |

#### 2.3.2 HTTP 代理测试

| 需求项 | 描述 |
|--------|------|
| **功能描述** | 测试节点代理是否可用 |
| **测试方式** | 通过代理访问测试 URL |
| **测试 URL** | http://www.gstatic.com/generate_204 |
| **并发数** | 20 |
| **超时时间** | 10 秒 |
| **业务规则** | 1. 根据节点类型构建代理客户端<br>2. 记录响应延迟<br>3. 延迟 < 5000ms 视为有效 |
| **验收标准** | 无效节点被过滤，有效节点记录延迟 |

## 4. 约束条件

### 4.1 技术约束

- Python 版本 >= 3.11
- GitHub Actions 免费额度限制（每月 2000 分钟）
- GitHub Pages 单文件大小限制（100 MB）

### 4.2 业务约束

- 节点测试仅用于验证有效性，不保证长期稳定
- 免费节点可能随时失效，需高频更新
- 遵守 GitHub 服务条款
