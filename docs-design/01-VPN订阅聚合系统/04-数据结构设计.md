# 数据结构设计

## 1. 数据存储方案

本项目采用**文件存储**方案，无需数据库。数据以文件形式存储在 `data/` 和 `output/` 目录中。

---

## 2. 文件存储结构

```
data/
├── sources/                      # 手动订阅文件
│   ├── custom_nodes.yaml        # YAML 格式节点
│   ├── custom_nodes.txt         # TXT 格式节点
│   └── custom_nodes_base64.txt  # Base64 格式节点
│
└── cache/                        # 缓存数据
    ├── telegram_messages.json   # Telegram 消息缓存
    └── geoip_db.mmdb            # GeoIP 数据库

output/
├── clash.yaml                    # Clash 订阅输出
├── v2ray.txt                     # V2Ray 订阅输出
├── singbox.json                  # Sing-box 订阅输出
├── nodes.json                    # 节点详细数据
└── stats.json                    # 统计信息
```

## 5. 文件读写规范

### 5.1 读操作

```python
import json
import yaml
from pathlib import Path

# 读取 JSON
def read_json(path: str) -> dict:
    file = Path(path)
    if not file.exists():
        return {}
    return json.loads(file.read_text(encoding="utf-8"))

# 读取 YAML
def read_yaml(path: str) -> dict:
    file = Path(path)
    if not file.exists():
        return {}
    return yaml.safe_load(file.read_text(encoding="utf-8"))

# 读取文本
def read_text(path: str) -> str:
    file = Path(path)
    if not file.exists():
        return ""
    return file.read_text(encoding="utf-8")
```

### 5.2 写操作

```python
import json
import yaml
from pathlib import Path

# 写入 JSON
def write_json(path: str, data: dict):
    Path(path).write_text(
        json.dumps(data, indent=2, ensure_ascii=False),
        encoding="utf-8"
    )

# 写入 YAML
def write_yaml(path: str, data: dict):
    Path(path).write_text(
        yaml.dump(data, allow_unicode=True, sort_keys=False),
        encoding="utf-8"
    )

# 写入文本
def write_text(path: str, content: str):
    Path(path).write_text(content, encoding="utf-8")
```

## 6. 数据生命周期

### 6.1 更新周期

| 数据类型 | 更新频率 | 保留时间 |
|---------|---------|---------|
| 订阅输出 | 每 6 小时 | 永久（GitHub Release 保留 30 天） |
| 节点缓存 | 每次更新 | 下次更新时覆盖 |
| Telegram 缓存 | 每次更新 | 1 小时过期 |
| GeoIP 数据库 | 每月 | 永久 |

### 6.2 数据清理

```python
def cleanup_old_data():
    """清理过期数据"""
    cache_dir = Path("data/cache")

    # 清理过期的 Telegram 缓存
    telegram_cache = cache_dir / "telegram_messages.json"
    if telegram_cache.exists():
        cache_time = telegram_cache.stat().st_mtime
        if time.time() - cache_time > 3600:  # 1 小时
            telegram_cache.unlink()
```

## 7. 数据备份

### 7.1 GitHub Release

每次更新自动创建 Release，保留历史版本：

```
v2026.06.14-08.00    # 最新版本
v2026.06.14-02.00    # 6 小时前
v2026.06.13-20.00    # 12 小时前
...
```

### 7.2 本地备份

可选配置本地备份：

```yaml
backup:
  enabled: true
  path: "backup/"
  keep_days: 30
```

## 8. 数据安全

### 8.1 敏感信息处理

- API Key 通过环境变量注入
- Telegram Session 文件加密存储
- 节点密码不明文记录日志

### 8.2 文件权限

```
data/cache/           # 读写
data/sources/         # 读写
output/               # 写入
config/               # 只读
```
