## 2. 模块设计

#### 2.1.1 模块职责

| 类名 | 职责 | 输入 | 输出 |
|------|------|------|------|
| Fetcher | 协调订阅抓取 | 订阅源配置 | 节点列表 |

## 3. 数据模型

### 3.1 Node 数据模型

```python
@dataclass
class Node:
    # 基础信息
    name: str              # 节点名称
    type: str              # 协议类型 (ss/vmess/vless/trojan/...)
    server: str            # 服务器地址
    port: int              # 端口号

    # 协议参数 (根据类型使用不同字段)
    cipher: str            # 加密方式
    password: str          # 密码
    uuid: str              # UUID
    alterId: int           # VMess alterId
    flow: str              # VLess flow
    network: str           # 传输协议 (tcp/ws/grpc)
    security: str          # 安全协议 (tls/reality)

    # TLS 参数
    sni: str               # SNI
    skip_cert_verify: bool # 跳过证书验证
    alpn: List[str]        # ALPN

    # WebSocket 参数
    ws_path: str           # WS 路径
    ws_headers: dict       # WS Headers

    # Reality 参数
    reality_public_key: str
    reality_short_id: str

    # Hysteria2 参数
    hysteria2_password: str
    hysteria2_obfs: str

    # 元信息
    country: str           # 国家代码
    source: str            # 来源
    latency: int           # 延迟 (ms)
    speed: float           # 速度 (MB/s)
    is_valid: bool         # 是否有效

    # 指纹 (用于去重)
    fingerprint: str
```

### 3.2 Source 数据模型

```python
@dataclass
class Source:
    name: str              # 订阅源名称
    type: str              # 类型 (github/local/telegram)
    enabled: bool          # 是否启用
    priority: int          # 优先级 (1-5)
    url: str               # URL (GitHub)
    path: str              # 文件路径 (Local)
    format: str            # 格式 (clash/base64/...)
    node_count: int        # 节点数量
    last_update: datetime  # 最后更新时间
    error: str             # 错误信息
```

## 4. 配置设计

### 4.1 全局配置 (config/settings.yaml)

```yaml
# 更新设置
update:
  interval: 6        # 小时
  retry: 3           # 重试次数
  timeout: 30        # 超时秒数

# 测试设置
testing:
  tcp:
    enabled: true
    timeout: 3
    concurrent: 100
  http:
    enabled: true
    timeout: 10
    concurrent: 20
    test_url: "http://www.gstatic.com/generate_204"

  max_latency: 5000  # 最大延迟 ms
  min_speed: 0.5     # 最小速度 MB/s

# 输出设置

  formats:
    - clash
    - v2ray
    - singbox
```

## 6. 部署架构

### 6.1 GitHub Actions 工作流

```yaml
name: Update Subscription

on:
  schedule:
    - cron: '0 */6 * * *'  # 每 6 小时
  workflow_dispatch:

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: python scripts/run.py
        env:
          TELEGRAM_API_ID: ${{ secrets.TELEGRAM_API_ID }}
          TELEGRAM_API_HASH: ${{ secrets.TELEGRAM_API_HASH }}
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./docs
```

### 6.2 GitHub Pages 目录结构

```
docs/
├── index.html           # 订阅页面
├── clash.yaml           # Clash 订阅
├── v2ray.txt            # V2Ray 订阅
├── singbox.json         # Sing-box 订阅
├── nodes.json           # 节点数据
└── stats.json           # 统计信息
```
