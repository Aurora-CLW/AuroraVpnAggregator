# 配置说明文档

## 1. 配置文件结构

```
config/
├── settings.yaml           # 全局配置
├── sources/               # 订阅源配置
│   ├── github.yaml       # GitHub 订阅源
│   ├── local.yaml        # 本地文件配置
│   └── telegram.yaml     # Telegram 频道配置
└── templates/             # 订阅模板
    ├── clash.yaml        # Clash 模板
    └── singbox.json      # Sing-box 模板
```

## 2. 全局配置 (settings.yaml)

### 2.1 完整配置示例

```yaml
# ==================== 更新设置 ====================
update:
  interval: 6        # 更新间隔（小时）
  retry: 3           # 失败重试次数
  timeout: 30        # 请求超时（秒）

# ==================== 节点测试设置 ====================
testing:
  # TCP 可达性测试
  tcp:
    enabled: true
    timeout: 3           # 超时时间（秒）
    concurrent: 100      # 并发数

  # HTTP 代理测试
  http:
    enabled: true
    timeout: 10          # 超时时间（秒）
    concurrent: 20       # 并发数
    test_url: "http://www.gstatic.com/generate_204"  # 测试 URL

  # 速度测试（可选）
  speed:
    enabled: false
    test_file: "https://speed.cloudflare.com/__down?bytes=10000000"
    max_time: 30         # 最大测试时间（秒）

# ==================== 节点过滤设置 ====================
filter:
  max_latency: 5000      # 最大延迟（毫秒）
  min_speed: 0.5         # 最小速度（MB/s）

  # 排除的国家（使用 ISO 3166-1 二位字母代码）
  exclude_countries: []
  # - "XX"  # 示例

  # 只包含的国家（空为不限制）
  include_countries: []
  # - "US"
  # - "JP"
  # - "HK"

  # 排除的关键词
  exclude_keywords:
    - "过期"
    - "失效"
    - "到期"
    - "测试"

# ==================== 去重设置 ====================
dedup:
  method: fingerprint    # 去重方式: fingerprint / server_port
  max_per_server: 10     # 相同服务器最大节点数

# ==================== 输出设置 ====================
output:
  formats:
    - clash
    - v2ray
    - singbox

  naming: "{country} {type} {latency}ms"  # 节点命名格式
  sort_by: latency        # 排序方式: latency / speed / country
  max_nodes: 500          # 最大节点数（0 为不限制）

# ==================== 日志设置 ====================
logging:
  level: INFO            # 日志级别: DEBUG / INFO / WARNING / ERROR
  file: logs/aurora.log  # 日志文件路径
  rotation: "1 day"      # 日志轮转
  retention: "7 days"    # 日志保留

# ==================== Telegram 设置 ====================
telegram:
  enabled: false         # 是否启用 Telegram
  api_id: ${TELEGRAM_API_ID}      # 从环境变量读取
  api_hash: ${TELEGRAM_API_HASH}  # 从环境变量读取
  session_path: data/cache/telegram_session  # 会话文件路径

# ==================== GitHub 设置 ====================
github:
  release: true          # 是否自动创建 Release
  keep_releases: 30      # 保留历史版本数

# ==================== 通知设置 ====================
notification:
  enabled: false

  # Telegram Bot
  telegram_bot:
    token: ${TG_BOT_TOKEN}
    chat_id: ${TG_CHAT_ID}

  # 企业微信
  wecom:
    webhook: ${WECOM_WEBHOOK}
```

### 2.2 配置项说明

#### 2.2.1 更新设置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `update.interval` | int | 6 | 更新间隔（小时） |
| `update.retry` | int | 3 | 失败重试次数 |
| `update.timeout` | int | 30 | 请求超时（秒） |

#### 2.2.2 测试设置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `testing.tcp.enabled` | bool | true | 是否启用 TCP 测试 |
| `testing.tcp.timeout` | int | 3 | TCP 超时（秒） |
| `testing.tcp.concurrent` | int | 100 | TCP 并发数 |
| `testing.http.enabled` | bool | true | 是否启用 HTTP 测试 |
| `testing.http.timeout` | int | 10 | HTTP 超时（秒） |
| `testing.http.concurrent` | int | 20 | HTTP 并发数 |
| `testing.http.test_url` | str | gstatic | 测试 URL |

#### 2.2.3 过滤设置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `filter.max_latency` | int | 5000 | 最大延迟（毫秒） |
| `filter.min_speed` | float | 0.5 | 最小速度（MB/s） |
| `filter.exclude_countries` | list | [] | 排除的国家 |
| `filter.include_countries` | list | [] | 只包含的国家 |
| `filter.exclude_keywords` | list | [] | 排除的关键词 |

#### 2.2.4 输出设置

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `output.formats` | list | [clash,v2ray,singbox] | 输出格式 |
| `output.naming` | str | {country} {type} {latency}ms | 命名格式 |
| `output.sort_by` | str | latency | 排序方式 |
| `output.max_nodes` | int | 500 | 最大节点数 |

## 3. 订阅源配置

### 3.1 GitHub 订阅源 (sources/github.yaml)

```yaml
type: github
enabled: true

sources:
  # 订阅源配置
  - name: "ermaozi_clash"                    # 名称（唯一标识）
    url: "https://raw.githubusercontent.com/ermaozi/get_subscribe/main/subscribe/clash.yml"
    format: clash                            # 格式: clash/base64/singbox
    priority: 1                              # 优先级 (1-5，1 最高)
    enabled: true                            # 是否启用

  - name: "mheidari98_base64"
    url: "https://raw.githubusercontent.com/mheidari98/.proxy/main/all"
    format: base64
    priority: 2
    enabled: true

  # 可以添加更多订阅源...
```

#### 配置项说明

| 配置项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `name` | str | ✅ | 订阅源名称（唯一标识） |
| `url` | str | ✅ | 订阅地址 |
| `format` | str | ✅ | 格式：clash/base64/singbox/surge |
| `priority` | int | ❌ | 优先级 1-5，默认 5 |
| `enabled` | bool | ❌ | 是否启用，默认 true |

### 3.2 本地文件配置 (sources/local.yaml)

```yaml
type: local
enabled: true

sources:
  # YAML 格式节点文件
  - name: "custom_yaml_nodes"
    path: "data/sources/custom_nodes.yaml"   # 文件路径（相对项目根目录）
    format: clash                            # 格式
    watch: true                              # 是否监控文件变更
    enabled: true

  # TXT 格式节点文件
  - name: "custom_txt_nodes"
    path: "data/sources/custom_nodes.txt"
    format: auto                             # auto 自动识别格式
    watch: true
    enabled: true

  # Base64 格式节点文件
  - name: "custom_base64_nodes"
    path: "data/sources/custom_nodes_base64.txt"
    format: base64
    watch: true
    enabled: true
```

#### 支持的文件格式

| 格式 | 说明 | 示例 |
|------|------|------|
| `clash` | Clash YAML 格式 | `proxies: [...]` |
| `base64` | Base64 编码的节点列表 | 编码后的 vmess://... |
| `auto` | 自动识别（URL 列表） | 每行一个节点 URL |

#### 手动添加节点示例

**YAML 格式** (`data/sources/custom_nodes.yaml`):

```yaml
proxies:
  - name: "我的节点 1"
    type: vmess
    server: my-server.com
    port: 443
    uuid: xxxx-xxxx-xxxx-xxxx
    network: ws
    tls: true
    ws-opts:
      path: /path

  - name: "我的节点 2"
    type: trojan
    server: my-trojan.com
    port: 443
    password: my-password
    sni: my-trojan.com
```

**TXT 格式** (`data/sources/custom_nodes.txt`):


### 3.3 Telegram 频道配置 (sources/telegram.yaml)

```yaml
type: telegram
enabled: false  # 默认禁用，配置 API 后启用

# Telegram API 配置
# 获取方式: https://my.telegram.org/apps
api_id: ${TELEGRAM_API_ID}
api_hash: ${TELEGRAM_API_HASH}
session_path: "data/cache/telegram_session"

channels:
  # 频道配置
  - name: "ExampleV2Ray"                     # 频道名称（自定义）
    channel_id: -1001234567890               # 频道 ID（必须为数字 ID）
    max_messages: 100                        # 抓取最近 N 条消息
    keywords:                                # 关键词过滤（可选）
      - "vmess://"
      - "vless://"
      - "trojan://"
    enabled: true

  - name: "FreeVPNNodes"
    channel_id: -1009876543210
    max_messages: 50
    enabled: true

# 消息解析设置
parsing:
  patterns:
    - "vmess://[A-Za-z0-9+/=]+"
    - "vless://[A-Za-z0-9\\-._~:/?#\\[\\]@!$&'()*+,;=%]+"
    - "trojan://[A-Za-z0-9\\-._~:/?#\\[\\]@!$&'()*+,;=%]+"
    - "ss://[A-Za-z0-9+/=\\-._~:/?#\\[\\]@!$&'()*+,;=%]+"

  cache:
    enabled: true
    path: "data/cache/telegram_messages.json"
    expire: 3600  # 缓存过期时间（秒）

# 限流设置
rate_limit:
  delay: 1          # 请求间隔（秒）
  batch_size: 20    # 每批消息数
```

#### 如何获取 Telegram API

1. 访问 https://my.telegram.org
2. 使用手机号登录 Telegram 账号
3. 进入 "API development tools"
4. 创建应用，获取 `api_id` 和 `api_hash`

#### 如何获取频道 ID

1. 将频道消息转发给 @userinfobot
2. 或使用 @getmyid_bot 获取频道 ID
3. 频道 ID 通常为负数，如 `-1001234567890`

## 7. 常见问题

### Q: 如何添加新的订阅源？

A: 编辑 `config/sources/github.yaml`，添加新的订阅源配置。

### Q: 如何禁用某个订阅源？

A: 将订阅源的 `enabled` 设置为 `false`。

### Q: 如何调整测试并发数？

A: 修改 `config/settings.yaml` 中的 `testing.tcp.concurrent` 和 `testing.http.concurrent`。

### Q: 如何修改更新频率？

A: 修改 `.github/workflows/update-subscription.yml` 中的 `cron` 表达式。
